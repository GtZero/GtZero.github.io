JDK8 新特性学习

Veröffentlicht am 2019-07-23 | Edited on 2019-12-19 | 次阅读

附: JDK8 新特性

[TOC]

JDK1.8及之后，可以在接口中创建实例方法，default域！

一.Lambda表达式

1. 函数式接口

接口中只有一个抽象方法的接口。可以使用注解@FunctionalInterface 修饰来检查是否是函数式接口

2. 功能

可替代对函数式接口创建的匿名内部类，增加代码可读性

  • jdk1.7及之前，匿名内部类中应用的局部变量，必须被final修饰(不可进行修改操作)。1.8及之后可省略，没加会默认添加
  • Lambda表达式中同

3. 基本语法

  • 操作符: ->
  • 左侧: Lambda表达式的参数列表
  • 右侧(Lambda体): Lambda表达式中所需执行的功能

4. 语法格式

只有一条语句时，大括号可以省略。有返回值的接口，默认返回这一条语句的返回值

  1. 接口无/多个参数、无返回值，语句多条
  1. 接口无/多个参数、有返回值，语句多条

5. 四大内置接口

  1. Consumer : 消费型接口。
    void accept(T t)
  2. Supplier : 供给型接口
    T get()
  3. Function<T, R> : 函数型接口
    R apply(T t)
  4. Predicate : 断言型接口
    boolean test(T t)

二.方法引用

1.功能

通过类名和方法名来定位一个静态方法或实例方法。用来简化替换lambda表达式的一种手段

2.注意项

  • Lambda体中调用方法的参数列表与返回类型，与函数式接口中抽象方法的函数列表和返回值一致(即Lambda体与接口方法一致)

3. 语法格式

3.1.普通方法

1. 对象::实例方法名

2. 类::静态方法名

3. 类::实例方法名
当要对参数的方法进行引用时使用。是用第一个参数的该方法

3.2.构造方法

1.对象构造

格式
ClassName :: new

构造方法

用带返回值的函数式接口，参数列表要一致！返回值为类名

无参构造
Supplier<User> sup = User::new;

带一个参数
Function<String, User> fun = User::new;

两个参数
BiFunction<String, String, User> bif = User::new;

2.数组构造

格式
Type[] :: new

示例
Function<Integer, String[]> fun2 = String[]::new;

数组构造需要指定大小，所以需要传入参数，使用Function接口

4. 补充: 19.12.14

  • lambda简化匿名内部类的代码
  • 方法引用简化lambda表达式，只是实现了流式接口里的方法
  • 方法引用没有显示写出对应的参数及返回值。对应的流式接口的参数，要和方法引用的参数对上

  • 关于类调用实例方法，是表明我要调用该类实例的这个实例方法。具体的调用者，是参数的那个对象

  • 在这里表明调用User类的prin实例方法

  • 实际上是调用传进来的User对象的prin方法

三.Stream流操作

1.简介

数据渠道，用于操作数据源(集合、数组等)所生成的元素序列

2.操作步骤

2.1. 创建流Stream

  1. 通过Collection系列集合提供的stream() 或 parallelStream()
  2. 通过Arrays中的静态方法stream() 获得数组流
  3. 通过Stream类中的静态方法of()
  4. 创建无限流。
    Stream<Integer> stream = Stream.iterate(0, (x) -> x + 2);

2.2.中间操作

返回值是Stream

注意
在进行终止操作时中间操作会一次性全部处理 — “惰性求值”

筛选

  • filter — 接收Lambda，从流中清除某些元素
  • limit — 截断流，使元素不能超过给定数量
  • skip(n) — 跳过流中的前n个元素
  • distinct — 筛选，通过流所生成元素的hashCode方法、equals方法去除重复元素

映射

  • map(Function f) — 接收Lambda，该函数会被应用到每个元素上。即该接口的传入参数即流中的数据，返回值即转换成的类型
  • flatMap — 接收一个函数作为参数，将流中的每个值都换成另一个流，然后将所有流连续成一个流

排序

  • sorted() — 自然排序
  • sorted(Comparator com) — 定制排序

2.3 终止操作

返回值是非Stream

查找and匹配

返回值为boolean

  • allMatch — 检查是否匹配所有元素
  • anyMatch — 检查是否至少匹配一个元素
  • noneMatch — 检查是否没有匹配所有元素
  • findFirst — 返回第一个元素

  • findAny — 返回当前流中的任意元素

  • count — 返回流中元素的总个数，返回值为Long

  • max — 返回流中最大值，返回值为Optional

  • min — 返回值中最小值 ，返回值同max

归约

  • reduce — 将流中元素反复结合

收集

  • collect(Collector c)。— 将流转换为其他形式 Collectors实用类提供了很多返回Collector的静态方法

总得来说就是Stream的API使用

四.Optional容器类

常用方法

  • Optional.of(T t) : 创建一个Optional实例。该对象中t为null，会抛出异常
  • Optional.empty() : 创建一个空的Optional实例
  • Optional.ofNullable : 若t不为null，创建Optional实例，否则创建空实例
  • isPresent() : 判断是否包含值
  • orElse(T t), 如果对象包含值，返回该值。写替代对象，没有包含则返回替代对象。避免空指针异常
  • orElseGet(Supplier s) : 如果调用对象包含值，返回该值，否则返回s获取的值
  • map(Function f) : 如果有值对其处理，并返回处理后的Optional，否则返回Optional.empty()
  • flatMap(Function f) : 与map类似，要求返回值必须是Optional
  • get方法，获得真实数据

个人常用方法

  • orElse(T t)，在Optional中没有对象时，拿这个默认对象
  • orElseGet(Supplier s), 在为null时，用流式接口拿默认值 (与orElse相比，这里能写自己的逻辑，如在什么条件下，返回啥默认值)
  • map、flatMap、filter进行过滤

Optional的正确使用姿势

  • 尽量不直接调用get、isPresent()方法，应该依赖于orElse、orElseGet、map这样的方法

四.其他特性

1.接口方法

接口中可以定义默认方法(default 方法)，与public 静态方法

默认方法 “类优先”原则: 若一个接口中定义了一个默认方法，而另外一个父类或接口中又定义了一个同名方法时，选择父类中的方法

两个接口都提供一个具有相同名称和参数列表的方法，必须覆盖该方法来解决冲突

2.新日期API

LocalDate、LocalTime、LocalDateTime类的实例是不可变对象(线程安全对象)，分别使用ISO-8601日历系统的日期、时间、日期和时间

3.注解

jdk8开始可以重复注解，且可以在类型前写注解

在创建重复注解时，需加上@Repeatable
